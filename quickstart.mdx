## Prerequisites

Before you begin, make sure you have:

  <Card title="Resend API key" icon="key">

</Warning>

## Installation

<Steps>
  <Step title="Install uv package manager">
    uv is the fastest way to manage Python dependencies. Install it with pip:
    
    ```bash
    pip install uv
    ```
  </Step>

  <Step title="Clone the repository">
    ```bash
    git clone https://github.com/Nneji123/reze-agent.git
    cd reze-agent
    ```
  </Step>

  <Step title="Install dependencies">
    Let uv handle all dependencies automatically:
    
    ```bash
    uv sync
    ```
    
    This creates a virtual environment and installs all required packages from `pyproject.toml`:
    - `fastapi>=0.104.0` - Web framework
    - `uvicorn[standard]>=0.24.0` - ASGI server
    - `pydantic-ai>=0.1.0` - AI agent framework
    - `memvid-sdk>=1.0.0` - RAG capabilities
    - `sqlalchemy[asyncio]>=2.0.0` - Database ORM
  </Step>

  <Step title="Configure environment variables">
    Copy the example environment file and fill in your API keys:
    
    ```bash
    cp env.example .env
    ```
    
    Edit `.env` with your actual values:
    
    ```bash .env
    # GLM 4.7 AI from z.ai
    GLM_API_KEY=your_glm_api_key_here
    GLM_BASE_URL=https://open.bigmodel.cn/api/paas/v4
    GLM_MODEL=glm-4.7
    
    # Resend.com
    RESEND_API_KEY=re_xxxxxxxxxxxx
    RESEND_FROM_EMAIL=noreply@yourdomain.com
    
    # Memvid RAG
    MEMVID_FILE_PATH=./memory.mv2
    
    # Database
    DATABASE_URL=sqlite+aiosqlite:///./database.db
    ```
    
    <Tip>
    Keep your `.env` file secure and never commit it to version control. The `.gitignore` file already excludes it.
    </Tip>
  </Step>

  <Step title="Populate the knowledge base">
    Load Resend documentation into Memvid for RAG capabilities:
    
    ```bash
    uv run python scripts/populate_memvid.py
    ```
    
    This creates a `memory.mv2` file containing embeddings of Resend documentation. The process takes about 1-2 minutes.
    
    <Info>
    This step is optional for sending emails, but required if you want to ask questions about Resend features.
    </Info>
  </Step>

  <Step title="Start the server">
    Launch Reze with a single command:
    
    ```bash
    uv run python main.py
    ```
    
    You should see output like:
    
    ```
    ================================================================================
    ðŸš€ Reze AI Agent Starting...
    ================================================================================
    Host: 0.0.0.0
    Port: 8000
    Debug Mode: True
    Log Level: INFO
    GLM Model: glm-4.7
    Memvid File: ./memory.mv2
    Database: sqlite+aiosqlite:///./database.db
    ================================================================================
    INFO:     Started server process [12345]
    INFO:     Waiting for application startup.
    INFO:     Application startup complete.
    INFO:     Uvicorn running on http://0.0.0.0:8000
    ```
  </Step>
</Steps>

## Send your first email

Now that Reze is running, let's send an email through the chat interface.

### Using the web interface

<Steps>
  <Step title="Open your browser">
    Navigate to [http://localhost:8000](http://localhost:8000)
    
    You'll see a clean chat interface powered by Reze.
  </Step>

  <Step title="Start a conversation">
    Type a message like:
    
    ```
    Send an email to john@example.com welcoming him to our platform
    ```
    
    Press Enter or click Send.
  </Step>

  <Step title="Provide missing details">
    Reze will ask clarifying questions:
    
    ```
    What should the subject line be?
    ```
    
    Respond with:
    
    ```
    Welcome to Our Platform!
    ```
    
    Reze will then ask:
    
    ```
    What content would you like in the email?
    ```
    
    Respond with:
    
    ```
    Hi John, welcome aboard! We're excited to have you.
    ```
  </Step>

  <Step title="Email sent!">
    Reze confirms the email was sent:
    
    ```
    âœ“ Email sent successfully! Email ID: 4ef2e3e1-a890-4c89-b3f1-9b4e5c6d7e8f
    It should be delivered shortly. Would you like me to check its delivery status?
    ```
  </Step>
</Steps>

### Using the API

You can also interact with Reze programmatically using the REST API:

<CodeGroup>
```bash cURL
curl -N -X POST http://localhost:8000/api/chat/message \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Send an email to john@example.com with subject Welcome and content Hi John!",
    "username": "demo_user"
  }'
```

```python Python
import httpx
import json

url = "http://localhost:8000/api/chat/message"
payload = {
    "message": "Send an email to john@example.com with subject Welcome and content Hi John!",
    "username": "demo_user"
}

with httpx.stream("POST", url, json=payload, timeout=30.0) as response:
    for line in response.iter_lines():
        if line.startswith("data: "):
            try:
                data = json.loads(line[6:])
                print(data.get("text", ""), end="", flush=True)
            except json.JSONDecodeError:
                continue
```

```javascript JavaScript
const url = 'http://localhost:8000/api/chat/message';
const payload = {
  message: 'Send an email to john@example.com with subject Welcome and content Hi John!',
  username: 'demo_user'
};

const response = await fetch(url, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify(payload)
});

const reader = response.body.getReader();
const decoder = new TextDecoder();

while (true) {
  const { done, value } = await reader.read();
  if (done) break;
  
  const chunk = decoder.decode(value);
  const lines = chunk.split('\n');
  
  for (const line of lines) {
    if (line.startsWith('data: ')) {
      const data = JSON.parse(line.slice(6));
      process.stdout.write(data.text || '');
    }
  }
}
```
</CodeGroup>

<Note>
The `-N` flag in cURL disables buffering for streaming responses. The `username` field isolates conversations per user.
</Note>

## Check email status

After sending an email, you can track its delivery status:

<CodeGroup>
```text Chat
Check the status of that email
```

```bash cURL
curl -N -X POST http://localhost:8000/api/chat/message \
  -H "Content-Type: application/json" \
  -d '{
    "message": "Check status of email 4ef2e3e1-a890-4c89-b3f1-9b4e5c6d7e8f",
    "username": "demo_user",
    "conversation_id": "previous_conversation_id"
  }'
```
</CodeGroup>

Reze responds with detailed status information:

```
Email 4ef2e3e1-a890-4c89-b3f1-9b4e5c6d7e8f status: DELIVERED
- Created: 2026-02-28T10:30:45Z
- Last event: delivered
âœ“ The email has been successfully delivered to the recipient.
```

<Info>
Reze remembers the context of your conversation. If you sent an email earlier, you can simply say "check the status of that email" without providing the ID.
</Info>

## Ask about Resend features

Leverage Reze's RAG capabilities to get answers about Resend:

<CodeGroup>
```text Question
What are the email size limits in Resend?
```

```bash cURL  
curl -N -X POST http://localhost:8000/api/chat/message \
  -H "Content-Type: application/json" \
  -d '{
    "message": "What are the email size limits?",
    "username": "demo_user"
  }'
```
</CodeGroup>

Reze searches the Resend documentation and provides an accurate answer based on the latest information.

## View API documentation

Reze includes interactive API documentation powered by FastAPI:

- **Swagger UI**: [http://localhost:8000/docs](http://localhost:8000/docs)
- **ReDoc**: [http://localhost:8000/redoc](http://localhost:8000/redoc)

<Warning>
API docs are only available when `DEBUG=True` in your `.env` file. They're disabled in production for security.
</Warning>
