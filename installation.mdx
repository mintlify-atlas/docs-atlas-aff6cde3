## Installation methods

Choose the installation method that best fits your workflow:

<Tabs>
  <Tab title="uv (recommended)">
    ### Install with uv

    uv is the fastest Python package manager and the recommended way to install Reze.

    <Steps>
      <Step title="Install uv">
        If you don't have uv installed, get it via pip:
        
        ```bash
        pip install uv
        ```
        
        Or using the standalone installer:
        
        ```bash
        curl -LsSf https://astral.sh/uv/install.sh | sh
        ```
      </Step>

      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/Nneji123/reze-agent.git
        cd reze-agent
        ```
      </Step>

      <Step title="Install dependencies">
        uv automatically creates a virtual environment and installs all dependencies:
        
        ```bash
        uv sync
        ```
        
        This reads `pyproject.toml` and installs:
        - FastAPI and Uvicorn for the web server
        - PydanticAI for agent framework
        - Memvid SDK for RAG capabilities
        - SQLAlchemy and aiosqlite for database
        - All other dependencies listed in src/api/app.py:1-63
      </Step>

      <Step title="Configure environment">
        Copy the example environment file:
        
        ```bash
        cp env.example .env
        ```
        
        Edit `.env` with your API keys (see [Configuration](#configuration) below).
      </Step>

      <Step title="Run the server">
        ```bash
        uv run python main.py
        ```
        
        The server starts on `http://0.0.0.0:8000` by default.
      </Step>
    </Steps>

    <Tip>
    uv is significantly faster than pip and automatically manages virtual environments. It's perfect for development and production.
    </Tip>
  </Tab>

  <Tab title="pip">
    ### Install with pip

    Standard installation using pip and virtual environments.

    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/Nneji123/reze-agent.git
        cd reze-agent
        ```
      </Step>

      <Step title="Create a virtual environment">
        ```bash
        python -m venv .venv
        ```
        
        Activate it:
        
        <CodeGroup>
        ```bash Linux/macOS
        source .venv/bin/activate
        ```
        
        ```bash Windows
        .venv\Scripts\activate
        ```
        </CodeGroup>
      </Step>

      <Step title="Install dependencies">
        ```bash
        pip install -e .
        ```
        
        This installs all dependencies from `pyproject.toml` in editable mode.
      </Step>

      <Step title="Configure environment">
        ```bash
        cp env.example .env
        ```
        
        Edit `.env` with your API keys (see [Configuration](#configuration) below).
      </Step>

      <Step title="Run the server">
        ```bash
        python main.py
        ```
      </Step>
    </Steps>
  </Tab>

  <Tab title="Docker">
    ### Install with Docker

    Run Reze in a containerized environment for easy deployment.

    <Steps>
      <Step title="Clone the repository">
        ```bash
        git clone https://github.com/Nneji123/reze-agent.git
        cd reze-agent
        ```
      </Step>

      <Step title="Create environment file">
        ```bash
        cp env.example .env
        ```
        
        Edit `.env` with your API keys (see [Configuration](#configuration) below).
      </Step>

      <Step title="Build the Docker image">
        ```bash
        docker build -t reze-agent .
        ```
      </Step>

      <Step title="Run the container">
        ```bash
        docker run -d \
          --name reze \
          -p 8000:8000 \
          --env-file .env \
          -v $(pwd)/database.db:/app/database.db \
          -v $(pwd)/memory.mv2:/app/memory.mv2 \
          reze-agent
        ```
        
        The `-v` flags mount volumes for persistent data (database and knowledge base).
      </Step>

      <Step title="View logs">
        ```bash
        docker logs -f reze
        ```
      </Step>
    </Steps>

    <Info>
    The Docker image is based on Python 3.12-slim for minimal size. Database and Memvid files are persisted via volumes.
    </Info>
  </Tab>
</Tabs>

## Configuration

Reze uses environment variables for configuration. All settings are defined in `src/config.py` and loaded from your `.env` file.

### Required variables

These variables must be set for Reze to function:

```bash .env
# GLM 4.7 AI from z.ai
# Get your API key from: https://open.bigmodel.cn/
GLM_API_KEY=your_glm_api_key_here
GLM_BASE_URL=https://open.bigmodel.cn/api/paas/v4
GLM_MODEL=glm-4.7

# Resend.com
# Get your API key from: https://resend.com/api-keys
RESEND_API_KEY=re_xxxxxxxxxxxx

# Default sender email (must be verified in Resend)
RESEND_FROM_EMAIL=noreply@yourdomain.com
```

<Warning>
Never commit your `.env` file to version control. Keep your API keys secure.
</Warning>

### Optional variables

These variables have sensible defaults but can be customized:

```bash .env
# Application Settings
DEBUG=True
HOST=0.0.0.0
PORT=8000
LOG_LEVEL=INFO

# Sentry Error Tracking (optional)
# Get your DSN from: https://docs.sentry.io/platforms/python/guides/fastapi/
SENTRY_DSN=your_sentry_dsn_here

# Memvid RAG Settings
MEMVID_FILE_PATH=./memory.mv2
MEMVID_INDEX_KIND=basic

# Resend API Base URL
RESEND_BASE_URL=https://api.resend.com

# Database Settings
DATABASE_URL=sqlite+aiosqlite:///./database.db
DATABASE_ECHO=false
```

### Environment variable reference

<AccordionGroup>
  <Accordion title="Application settings">
    | Variable | Default | Description |
    |----------|---------|-------------|
    | `DEBUG` | `True` | Enable debug mode (shows API docs) |
    | `HOST` | `0.0.0.0` | Server host address |
    | `PORT` | `8000` | Server port number |
    | `LOG_LEVEL` | `INFO` | Logging level (DEBUG, INFO, WARNING, ERROR) |
    | `SENTRY_DSN` | `""` | Sentry DSN for error tracking |
  </Accordion>

  <Accordion title="GLM AI settings">
    | Variable | Default | Description |
    |----------|---------|-------------|
    | `GLM_API_KEY` | Required | GLM 4.7 API key from z.ai |
    | `GLM_BASE_URL` | `https://open.bigmodel.cn/api/paas/v4` | OpenAI-compatible API endpoint |
    | `GLM_MODEL` | `glm-4.7` | Model name (glm-4.7, glm-4, etc.) |
    
    <Tip>
    The GLM API is OpenAI-compatible, so you can easily switch to other providers by changing the base URL and API key.
    </Tip>
  </Accordion>

  <Accordion title="Resend settings">
    | Variable | Default | Description |
    |----------|---------|-------------|
    | `RESEND_API_KEY` | Required | Resend API key |
    | `RESEND_FROM_EMAIL` | Required | Default sender email (must be verified) |
    | `RESEND_BASE_URL` | `https://api.resend.com` | Resend API base URL |
    
    <Warning>
    Your sender email must be from a verified domain in your Resend account. See [Resend's documentation](https://resend.com/docs/dashboard/domains/introduction) for domain verification.
    </Warning>
  </Accordion>

  <Accordion title="Memvid RAG settings">
    | Variable | Default | Description |
    |----------|---------|-------------|
    | `MEMVID_FILE_PATH` | `./memory.mv2` | Path to Memvid knowledge base file |
    | `MEMVID_INDEX_KIND` | `basic` | Index type (basic or advanced) |
    
    <Info>
    The `memory.mv2` file stores embeddings of the Resend documentation. It's created when you run `scripts/populate_memvid.py`.
    </Info>
  </Accordion>

  <Accordion title="Database settings">
    | Variable | Default | Description |
    |----------|---------|-------------|
    | `DATABASE_URL` | `sqlite+aiosqlite:///./database.db` | SQLAlchemy database URL |
    | `DATABASE_ECHO` | `false` | Enable SQL query logging |
    
    For PostgreSQL in production:
    ```bash
    DATABASE_URL=postgresql+asyncpg://user:password@localhost:5432/reze
    ```
    
    <Tip>
    SQLite is perfect for development and small deployments. For production with multiple instances, use PostgreSQL.
    </Tip>
  </Accordion>
</AccordionGroup>

## Populate the knowledge base

To enable RAG capabilities, populate Memvid with Resend documentation:

<Steps>
  <Step title="Run the populate script">
    <CodeGroup>
    ```bash uv
    uv run python scripts/populate_memvid.py
    ```
    
    ```bash pip
    python scripts/populate_memvid.py
    ```
    </CodeGroup>
  </Step>

  <Step title="Wait for completion">
    The script scrapes Resend's documentation and creates embeddings. This takes 1-2 minutes:
    
    ```
    INFO: Fetching Resend documentation...
    INFO: Processing 45 pages...
    INFO: Creating embeddings...
    INFO: Memvid knowledge base created at ./memory.mv2
    INFO: Stats: 45 documents, 2.3MB
    ```
  </Step>

  <Step title="Verify">
    Check that `memory.mv2` was created:
    
    ```bash
    ls -lh memory.mv2
    ```
  </Step>
</Steps>

<Note>
This step is optional if you only want to send emails. It's required for asking questions about Resend features.
</Note>

## Troubleshooting

<AccordionGroup>
  <Accordion title="Import errors or missing dependencies">
    Make sure all dependencies are installed:
    
    ```bash
    uv sync
    # or
    pip install -e .
    ```
    
    Verify you're using Python 3.12 or higher:
    
    ```bash
    python --version
    ```
  </Accordion>

  <Accordion title="API key errors">
    Double-check your `.env` file:
    
    ```bash
    cat .env | grep -E "GLM_API_KEY|RESEND_API_KEY"
    ```
    
    Make sure there are no extra spaces or quotes around the keys.
  </Accordion>

  <Accordion title="Database connection errors">
    Check that the database file is writable:
    
    ```bash
    touch database.db
    ls -l database.db
    ```
    
    For PostgreSQL, verify the connection string:
    
    ```bash
    psql $DATABASE_URL -c "SELECT 1;"
    ```
  </Accordion>

  <Accordion title="Port already in use">
    Change the port in your `.env` file:
    
    ```bash
    PORT=8001
    ```
    
    Or specify it when running:
    
    ```bash
    PORT=8001 uv run python main.py
    ```
  </Accordion>

  <Accordion title="Memvid file not found">
    Run the populate script:
    
    ```bash
    uv run python scripts/populate_memvid.py
    ```
    
    Or disable RAG by commenting out the Memvid initialization in `src/api/app.py`.
  </Accordion>
</AccordionGroup>

## Upgrading

To upgrade Reze to the latest version:

<Steps>
  <Step title="Pull latest changes">
    ```bash
    git pull origin main
    ```
  </Step>

  <Step title="Update dependencies">
    <CodeGroup>
    ```bash uv
    uv sync
    ```
    
    ```bash pip
    pip install -e . --upgrade
    ```
    </CodeGroup>
  </Step>

  <Step title="Check for breaking changes">
    Review the changelog and update your `.env` file if needed:
    
    ```bash
    diff env.example .env
    ```
  </Step>

  <Step title="Restart the server">
    ```bash
    uv run python main.py
    ```
  </Step>
</Steps>
